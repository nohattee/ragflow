Utilities

This page documents the utility functions and helpers provided by RAGFlow.

Helpers

Utility functions for RAGFlow.

This module provides helper functions for common operations when working with RAGFlow, designed to reduce boilerplate code and improve developer experience.

ragflow.utils.helpers.load_text_files(directory_path, file_pattern='*.txt', encoding='utf-8', recursive=False, metadata_fn=None)[source]

Load text files from a directory into Document objects.

Parameters:

  • directory_path (str) – Path to the directory containing text files

  • file_pattern (str) – Glob pattern to match files (default: “*.txt”)

  • encoding (str) – File encoding to use (default: “utf-8”)

  • recursive (bool) – Whether to search subdirectories recursively (default: False)

  • metadata_fn (Callable[[str], Dict[str, Any]] | None) – Optional function to generate metadata from a file path Function signature should be: (filepath: str) -> Dict[str, Any]

Returns:

List of Document objects, one for each file

Raises:

DocumentProcessingError – If there’s an issue reading or processing files

Return type:

List[Document]

Examples:

Basic usage with default settings:

docs = load_text_files("./data")
# Assuming 'pipeline' is an initialized RAGPipeline object
# pipeline.add_documents(docs)

With custom metadata function:

import os


def create_metadata(filepath):
    filename = os.path.basename(filepath)
    return {"filename": filename, "date_added": "2025-05-07"}


docs = load_text_files("./data", metadata_fn=create_metadata)
# pipeline.add_documents(docs)
ragflow.utils.helpers.create_pipeline_from_env(collection_name='ragflow', persist_directory='./', env_var='GEMINI_API_KEY', **kwargs)[source]

Create a DefaultRAGPipeline using an API key from an environment variable.

This is a convenience function for the common case of creating a pipeline with an API key stored in an environment variable.

Parameters:

  • collection_name (str) – Name of the ChromaDB collection to use

  • persist_directory (str) – Directory where ChromaDB will store its data

  • env_var (str) – Name of the environment variable containing the API key

  • **kwargs – Additional parameters to pass to DefaultRAGPipeline constructor

Returns:

A configured DefaultRAGPipeline instance

Raises:

ConfigurationError – If the environment variable is not set

Return type:

DefaultRAGPipeline

Examples:

# Assuming GEMINI_API_KEY is set in the environment
pipeline = create_pipeline_from_env()
pipeline.add_texts(["Document 1", "Document 2"])
answer = pipeline.query("What is in the documents?")
ragflow.utils.helpers.format_sources(result, include_content=False, max_chars=100)[source]

Format the sources from a query_with_sources result into a readable string.

Parameters:

  • result (Dict[str, Any]) – The result dictionary from query_with_sources()

  • include_content (bool) – Whether to include the content of each source (default: False)

  • max_chars (int) – Maximum characters to show for each source content (default: 100)

Returns:

A formatted string with source information

Return type:

str

Examples

`python result = pipeline.query_with_sources("What is RAGFlow?") print(result["answer"]) print("\nSources:") print(format_sources(result)) `
